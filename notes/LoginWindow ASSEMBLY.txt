## **5. Observations / Notes**

* **Display sleep** path and **screensaver** path are separate; previous edits only partially disabled locking.
* The **real control point** is the lock flag in:

```
-[LWScreenLock setScreenIsLocked:]  → byte at [rax+0x28]
```

* Runtime patching recommendation:

```lldb
expr *(uint8_t*)ADDR = 0x00
```

This forces lock state = 0, letting all paths (screensaver, sleep, hot corner) run without triggering immediate lock.

## **6. Summary of Lessons Learned**

1. Mid-method instruction patches shift bytes if not applied carefully → always use per-byte `expr` writes.
2. Hot corner triggers screensaver → different path than display sleep → patching only one path leaves other behaviors intact.
3. Final authoritative patch must target **lock state variable** instead of intermediate instructions.
4. Use this file as the canonical reference for scripting LaunchAgent/Daemon patches.
